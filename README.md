# BatchLauncher — 批量启动器


## 项目概述

BatchLauncher 是一个基于 **WPF (.NET 8)** 的 Windows 桌面应用程序，用于管理和批量启动多个软件。用户可以添加软件（支持路径和启动参数），勾选后一键启动选中项或全部启动，配置会持久化到本地。
主要是为了方便工具间的一键启动，摆脱经常找软件图标启动的痛苦。
其实就是基于命令行启动改成图形化启动。

## 核心功能

### 1. 软件管理

- **添加软件**：弹出对话框，填写软件名称、程序路径（支持浏览选择 `.exe`/`.lnk` 文件）、启动参数（可选）。选择文件后自动填充软件名称。
- **删除软件**：勾选要删除的软件，点击"删除选中"按钮即可移除。
- **软件列表展示**：以 ListView 呈现，列包含复选框、软件名称、路径、启动参数（也就是用命令行启动时那种带的参数）。

### 2. 批量启动

- **启动选中**：仅启动勾选了复选框的软件，逐个启动（每个间隔 500ms），实时显示进度。
- **启动全部**：启动列表中的所有软件，无需逐个勾选。
- **路径校验**：启动前检查文件/目录是否存在，路径无效时弹出提示并跳过。
- **启动参数支持**：可附加命令行参数传递给目标程序。

### 3. 选择控制

- **全选/取消全选**：顶部"全选"复选框支持三态（全选/全不选/部分选中），与列表中的逐项复选框双向同步。

### 4. 配置持久化

- **自动保存**：每次添加或删除软件后自动保存配置。
- **自动加载**：程序启动时从本地读取上次保存的软件列表。
- **存储位置**：`%LocalAppData%\BatchLauncher\config.json`，使用 JSON 格式，缩进美化，字段忽略大小写。

### 5. 状态反馈

- 底部状态栏实时显示操作结果（如"已添加新软件"、"正在启动第 X/N 个"、"完成 — 成功 X 个, 失败 Y 个"等）。

## 关键文件说明

### `Models/SoftwareInfo.cs`

软件信息数据模型，实现 `INotifyPropertyChanged` 以支持 UI 双向绑定。字段：

| 属性         | 类型     | 说明                         | 是否持久化 |
|-------------|----------|------------------------------|-----------|
| Name        | string   | 软件显示名称                  | ✅ 是      |
| Path        | string   | 程序文件路径                  | ✅ 是      |
| Arguments   | string   | 启动参数                      | ✅ 是      |
| IsSelected  | bool     | 是否被勾选（仅 UI 状态）       | ❌ 否（JsonIgnore） |

### `Services/ConfigService.cs`

静态配置服务类，负责软件列表的读写：

- **Load()**：从 `config.json` 反序列化加载列表，文件不存在或读取失败时返回空列表。
- **Save()**：将列表序列化为 JSON 写入文件，自动创建目录。保存失败时输出调试日志，不崩溃。

### `MainWindow.xaml.cs`

主窗口逻辑，核心职责：

- 加载/保存软件列表
- 响应按钮点击（添加、启动选中、启动全部、删除选中、全选）
- 通过 `Process.Start` 启动软件（`UseShellExecute = true`，支持 `.exe`、`.lnk` 等）
- 逐个启动时间隔 500ms，异步执行，不阻塞 UI

### `AddSoftwareDialog.xaml.cs`

添加软件对话框：

- 浏览选择文件（支持 `.exe` 和 `.lnk`）
- 选择文件后自动填充软件名称（去扩展名）
- 确定/取消按钮，输入校验
- 通过 `Result` 属性返回 `SoftwareInfo` 对象

## 使用流程

1. **启动程序** → 自动加载上次保存的软件列表
2. **点击"添加软件"** → 填写名称、选择路径、输入参数 → 确定
3. **勾选要启动的软件**（或使用"全选"）
4. **点击"启动选中"或"启动全部"** → 程序逐个启动，状态栏显示进度
5. 不再需要的软件 → 勾选后点击"删除选中"
6. **关闭程序** → 配置已自动保存，下次打开恢复
